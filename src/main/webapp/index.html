About the platform

Built for the way
logistics actually moves.

CargoTrack is an enterprise shipment-management platform designed around three principles: every action is auditable, every shipment is trackable, and every user only sees what they own. No surveillance theatre — just the visibility operators need.

Most cargo systems are spreadsheets pretending to be software. They lose track of who booked what, leak customer data across accounts, and treat security as a checkbox. CargoTrack treats security as the foundation.

Every shipment carries a tracking number. Every user has exactly the access their role requires. Every login goes through brute-force protection, every input is validated, every secret lives in environment variables — never in code.

The result: a platform a junior developer can audit, a senior developer can extend, and a customer can trust.

◆
Security-first
JWT auth, role-based access, sliding-window rate limiting, brute-force throttling, and email verification — all wired in by default.
↻
Built to scale
Stateless JAX-RS resources, JTA-managed transactions, paginated queries — every endpoint designed to survive real traffic.
∴
Auditable by design
Immutable tracking events form a per-cargo audit trail. Customer username on every shipment. No silent state changes.
What you get

Nine security layers,
zero enterprise overhead.

Every feature below ships with the platform. No add-ons, no surprise modules, no premium tiers. The whole stack runs on a single Jakarta EE container against a single PostgreSQL database.

🔐
JWT Authentication
HS256-signed JSON Web Tokens issued at login, validated on every protected request. Forge-resistant by design — change a single byte, signature fails.
jjwt 0.12 · 8h sessions
⚡
Role-Based Access
Three roles (CUSTOMER, OPERATOR, ADMIN), enforced at the filter layer. Customers only see their own shipments — full stop. Operators manage cargo state. Admins manage everything.
Filter priority 2000 · RBAC
✉
Email Verification
New accounts cannot log in until they confirm their address. The verify-then-flag-check ordering prevents login from becoming an account-enumeration oracle.
24h tokens · Single-use
⛔
Per-IP Rate Limiting
Sliding-window limiter at filter priority 100 — runs before authentication so abusive clients can't burn signature-verification CPU. Returns 429 with Retry-After.
100 req / 60s · Per IP
🛡
Brute-Force Throttle
Login-specific lockout keyed by (IP, username). Five failures in 15 minutes locks that bucket for the rest of the window. Constant-time response timing — no enumeration leak.
5 fails / 15m · Per (IP, user)
✓
Input Validation
Bean Validation on every DTO and resource parameter. UN/LOCODE patterns, weight bounds, tracking-number format, page/size caps — invalid requests die at the framework boundary.
Jakarta Bean Validation 3.0
From booking to delivery

Four steps
between origin and arrival.

The whole shipment lifecycle from a customer's perspective. Each step is its own API call, each transition is logged, each state change is auditable.

01
Register & Verify
Sign up with email + password. Click the link mailed to your inbox to confirm. Login is blocked until you do.
→
02
Book a Shipment
Origin port, destination port, weight, description. The system generates a unique CGO-XXXXXXXXXX tracking number and emails you a confirmation.
→
03
Track in Real Time
As operators record events (picked up, departed, arrived, delivered) the cargo's status advances automatically. The same tracking page works for guests via a public link.
→
04
Delivered
A DELIVERED event at the destination port closes the shipment. The full event history stays attached to the cargo as an immutable audit trail.
Access model

Three roles.
Each sees exactly what it should.

Role enforcement happens in RoleFilter — a JAX-RS container filter at priority 2000. No business-logic check is bypassable; the filter runs before any resource method.

Customer
Self-serve
  • Book new shipments
  • List their own shipments
  • Track any shipment by number
  • Update or cancel shipments
  • View other customers' data
  • Manage users or locations
Operator
Cargo control
  • All customer permissions
  • List every shipment in the system
  • Update cargo details & status
  • Record tracking events
  • View analytics dashboards
  • Permanently delete cargos
Admin
Full access
  • All operator permissions
  • Cancel shipments
  • Permanently delete cargos
  • Manage users (deactivate, change role)
  • Manage location catalog
  • Full system observability
